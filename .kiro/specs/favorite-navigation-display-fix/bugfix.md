# 收藏导航显示Bug修复需求文档

## 简介

本文档描述收藏页面导航功能的显示bug。用户在收藏页面点击导航按钮后，虽然后台逻辑正常执行（路径规划成功、绘制命令已发送），但界面没有正确响应：未切换到导航Tab，地图上也未显示路径。

## Bug分析

### 当前行为（缺陷）

1.1 当用户在收藏页面点击某个收藏地点的"导航"按钮时，系统返回主页面但未切换到导航Tab（停留在之前的Tab）

1.2 当用户在收藏页面点击某个收藏地点的"导航"按钮时，地图上未显示导航路径（虽然后台路径规划已成功）

1.3 当用户在收藏页面点击某个收藏地点的"导航"按钮时，导航目的地参数可能因时序问题未被正确传递到导航模块

### 期望行为（正确）

2.1 当用户在收藏页面点击某个收藏地点的"导航"按钮时，系统应自动切换到导航Tab（index=1）

2.2 当用户在收藏页面点击某个收藏地点的"导航"按钮时，地图应显示从当前位置到目的地的导航路径

2.3 当用户在收藏页面点击某个收藏地点的"导航"按钮时，导航模块应接收到正确的目的地参数并启动导航

### 不变行为（回归预防）

3.1 当用户在主页面的导航Tab中直接搜索并导航时，系统应继续正常显示导航路径

3.2 当用户在主页面通过POI搜索结果点击导航时，系统应继续正常切换到导航Tab并显示路径

3.3 当用户在收藏页面点击"删除"按钮时，系统应继续正常删除收藏项而不影响导航功能

3.4 当用户在收藏页面点击"返回"按钮时，系统应继续正常返回主页面并保持原有Tab状态
